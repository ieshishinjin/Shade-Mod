# 世界等级（World Level）系统

## 设计理念

世界等级是一个全局难度缩放系统，灵感来源于《原神》的"世界等级"机制。不同于传统 RPG 的固定区域等级或动态难度缩放，世界等级与**玩家经验等级**直接挂钩——服务器上最高经验等级的玩家决定当前世界等级。

这意味着：当玩家通过正常游戏积累经验时，世界会自然而然地变得更具挑战性，同时掉落更好的战利品。

## 等级划分

| 世界等级 | 所需最高玩家经验等级 | 颜色 |
|---------|-------------------|------|
| 0 | 0 | 灰色 (§7) |
| 1 | 10 | 深灰 (§8) |
| 2 | 20 | 白色 (§f) |
| 3 | 30 | 黄色 (§e) |
| 4 | 40 | 浅蓝 (§b) |
| 5 | 50 | 粉色 (§d) |
| 6 | 60 | 金色 (§6) |

阈值表：`{ 0, 10, 20, 30, 40, 50, 60 }`

玩家经验等级 = 当前经验条显示的等级（非总累积经验）。

## 视觉表现

### TAB 列表显示

按 TAB 打开玩家列表时，顶部会显示世界等级标题，每个玩家名字后面显示世界等级数字。

- **标题**：`✦ 世界等级 X ✦`（金色粗体）
- **玩家分数**：玩家名后显示当前世界等级数字（取代原版多人联机的网络延迟图标）

### 命令

- `/worldlevel` — 查看当前世界等级、最高玩家等级、怪物血量加成、宝箱品质

查看格式示例：
```
✦ 世界等级: WL.3  (最高玩家等级: 45)
  怪物血量: +60%  宝箱品质: ✦ 世界等级 3
```

## 对怪物的影响

当怪物生成时，根据当前世界等级应用以下状态效果：

| 世界等级 | 效果 | 等级 |
|---------|------|------|
| 1+ | 伤害吸收 (Absorption) | WL - 1 |
| 2+ | 抗性提升 (Resistance) | WL - 2 |
| 4+ | 力量提升 (Strength) | WL - 4 |
| 6+ | 生命恢复 (Regeneration) | I |

所有效果无限持续时间（实际设定 1,000,000 tick）。

## 对宝箱的影响

击败据点后生成的宝箱等级受世界等级加成：

```
最终等级 = min(原始等级索引 + WL / 2, 最高等级)
```

例如 WL=3 时，原本 BASIC 级宝箱（索引 0）会提升至 0 + 3/2 = 1（COMMON 级）。

## 技术实现

### 核心类设计

```
WorldLevel (工具类)
├── getLevel()          — 计算当前世界等级
├── getLevelFromPlayers() — 从玩家列表计算
├── getName()           — 获取等级显示名称
├── applyScaling()      — 给怪物施加状态效果
├── updateTabHeader()   — 更新 TAB 列表显示
├── getMaxLevel()       — 获取最高玩家经验等级
└── getMaxLevelFromPlayers() — 从玩家列表获取最高等级
```

### 计分板集成

使用 `DisplaySlot.LIST` 槽位显示世界等级。计分板目标命名为 `shade_wl`，启动时创建一次，之后只更新显示名称和分数值，不重建目标以避免闪屏。

### 注意

- 世界等级仅基于**在线玩家**的最高经验等级。如果最高等级玩家下线，等级可能下降。
- 世界等级为 0 时，怪物不获得任何状态效果（原生强度）。
- 所有状态效果含粒子效果为 false（`false, false`），避免屏幕被粒子淹没。
